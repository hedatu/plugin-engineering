# Paywall UX Rules

## Core Rules
- The user must be able to try the core value for free.
- Pro features must be described clearly in both the listing and the UI.
- If the core action has a free limit, both the listing and the popup must disclose it.
- Do not use fake countdown timers.
- Do not use misleading button labels.
- Prefer `lifetime access to the current major version` instead of vague forever claims.
- The external payment page must disclose that the seller is not Google.
- `local-only` claims must remain strictly true.
- If license verification requires a network request, say so in the UI.

## Recommended Messaging
- Free tier: explain the exact free-action allowance.
- Paid tier: explain the lifetime unlock price and what changes after activation.
- Failure states: show invalid, expired, revoked, and offline-grace messages plainly.
- Payment boundary: remind the user that checkout happens outside the extension.
