LeadFill One Profile Privacy

The extension stores one local profile in Chrome storage so the user can fill matching visible fields on the current page.

No remote sync, analytics, or third-party requests are used.

By default the extension does not overwrite fields that already contain values unless the user explicitly enables overwrite in the popup.

The extension only injects a fill script into the active tab when the user clicks Fill Current Page.

The extension enforces a free limit of 10 fills before showing an upgrade prompt.

Form data stays local to the extension and current tab. No upload of form content and no cloud sync are used.

Email login, checkout, entitlement refresh, and usage metering are routed through the background service worker with public pay-site config only.

The extension never stores payment provider secrets, service-role keys, or merchant secrets.

successUrl does not unlock Pro locally. Only webhook-confirmed active entitlement unlocks Pro.

Local-only form data. No upload of form content. Payment handled on external secure page. Membership unlocks after webhook-confirmed entitlement.
